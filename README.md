# Squad Clan Discord Bot

Многофункциональный Discord-бот для управления кланом в игре **Squad**.  
Бот состоит из двух независимых систем:

1. **Система опыта и званий (XP & Ranks)** — автоматизирует выдачу званий на основе ролей-ачивок.
2. **Трекер активности (Activity Tracker)** — отслеживает, сколько времени участники клана проводят в игре, автоматически разделяя время на три категории: **Бой** (Battle), **Сидинг** (Seeding) и **Тренировки** (Training). Ведёт детальный учёт по серверам, составляет топы игроков и поддерживает **фильтрацию статистики по периодам** (день, неделя, месяц, год).

## Стек технологий

- **Python 3.9+**
- **discord.py** (фреймворк для Discord API)
- **MongoDB** + **Motor** / **Pymongo** (асинхронная и синхронная работа с БД)

## 1. Настройка в Discord Developer Portal

Перед запуском бота его необходимо правильно настроить на [портале разработчиков](https://discord.com/developers/applications).

1. Создайте новое приложение.
2. Во вкладке **Bot**:
   - Отключите ползунок `Requires OAuth2 Code Grant`.
   - В разделе **Privileged Gateway Intents** ОБЯЗАТЕЛЬНО включите два ползунка:
     - `Server Members Intent` (нужен для отслеживания изменения ролей-ачивок).
     - `Presence Intent` (нужен для отслеживания статуса "Играет в Squad").
3. Во вкладке **OAuth2 → URL Generator**:
   - В блоке **Scopes** выберите: `bot` и `applications.commands`.
   - В блоке **Bot Permissions** выберите: `Administrator` (или вручную выдайте `Manage Roles`, `Send Messages`, `Embed Links`, `Use Slash Commands`).
4. Скопируйте сгенерированную ссылку в самом низу и добавьте бота на сервер.

> **КРИТИЧЕСКИ ВАЖНО:** После добавления бота зайдите в Настройки Сервера → Роли. Переместите роль бота **В САМЫЙ ВЕРХ СПИСКА**. Если роль бота окажется ниже ролей званий, бот не сможет их выдавать (Discord вернёт ошибку `Forbidden`).

## 2. Установка и запуск

1. Убедитесь, что установлен Python и локально запущена MongoDB (стандартный порт `27017`).
2. Откройте терминал в папке проекта и установите зависимости:
   ```bash
   pip install discord.py motor pymongo python-dotenv
   ```
3. Создайте файл `.env` в корневой папке проекта:
   ```env
   DISCORD_TOKEN=ВАШ_ТОКЕН_БОТА
   MONGO_URI=mongodb://localhost:27017/
   STEAM_API_KEY=ВАШ_КЛЮЧ_STEAM_API

   # ID роли клана (бот отслеживает активность только пользователей с этой ролью)
   CLAN_ROLE_ID=111111111111111111

   # ID ролей администраторов бота (через запятую без пробелов)
   ADMIN_ROLE_IDS=222222222222222222,333333333333333333
   ```
4. Запустите бота:
   ```bash
   python main.py
   ```

## 3. Массовый импорт состава (import_users.py)

Чтобы не привязывать каждого бойца вручную, предусмотрен скрипт для заливки таблицы из Excel/Google Sheets.

1. Откройте файл `users_data.txt`.
2. Вставьте данные бойцов. Формат: 4 колонки, разделённые `Tab` (стандарт при копировании из таблиц):
   `[Ник в Squad]   [Steam ID]   [Discord Ник]   [Discord ID]`
3. Запустите скрипт:
   ```bash
   python import_users.py
   ```

Скрипт свяжет Discord ID с игровым ником и Steam ID. База готова к трекингу.

## 4. Как это работает (Архитектура)

### Система опыта и званий (XP System)

Опыт не хранится в виде числа, а считается «на лету» из суммарной ценности ролей-ачивок, выданных пользователю.

1. Бот реагирует на выдачу/снятие роли модератором (событие `on_member_update`).
2. Проверяет наличие `CLAN_ROLE_ID` у пользователя.
3. Встроена **перекрёстная защита**: роль не может быть одновременно ачивкой и званием.
4. При удалении ачивки из БД бот **автоматически снимает её со всех участников** и пересчитывает их звания по цепочке.
5. Ачивки поддерживают **категоризацию** — можно группировать достижения по разделам (например, «Боевые», «Организационные») и фильтровать список командой `/achievements_list`.

### Трекер активности (Squad Tracker)

Трекер работает по принципу *Stateless 1-minute loop* с использованием *BulkWrite* и системы **Daily Buckets** (ежедневных корзин). В цикл встроены асинхронные паузы для защиты от зависаний при большом онлайне.

1. Каждую минуту бот проверяет Rich Presence тех пользователей, у которых есть `CLAN_ROLE_ID` и которые занесены в БД.
2. Разделяет время по трём категориям:
   - **Тренировки** — имя сервера содержит `[FREE] Zone`; имя нормализуется в `[FREE] Zone - Training Server` (проверяется первой, до остальных правил).
   - **Сидинг** — меньше 50 человек на сервере.
   - **Бой** — 50 и более человек на сервере.
3. Сохраняет данные в MongoDB двумя независимыми потоками: в глобальную коллекцию (за всё время) и в ежедневную (ключ `Steam ID + Дата`). Это гарантирует сохранность данных при сбоях и позволяет мгновенно генерировать статистику за любой период через *MongoDB Aggregation Pipelines*.

## 5. Список команд (Slash Commands)

### Команды Администратора

*Доступны только пользователям с ролями, указанными в `ADMIN_ROLE_IDS` в `.env`.*

#### Управление ачивками

| Команда | Параметры | Описание |
|---|---|---|
| `/add_achievement` | `роль`, `xp`, `описание`, `[category]` | Регистрирует роль как ачивку с ценностью опыта. Параметр `category` необязателен, по умолчанию `Без категории`. |
| `/edit_achievement` | `роль`, `new_xp`, `new_description`, `[new_category]` | Изменяет XP и описание ачивки. Если передан `new_category` — обновляет и категорию. |
| `/set_achievement_category` | `роль`, `category` | Меняет **только категорию** существующей ачивки, не затрагивая XP и описание. Удобно для массовой реорганизации без правки других полей. |
| `/remove_achievement` | `роль` | Удаляет ачивку из БД и **физически снимает роль со всех участников**. |

#### Управление званиями

| Команда | Параметры | Описание |
|---|---|---|
| `/add_rank` | `роль`, `required_xp` | Регистрирует роль как звание с порогом выдачи. |
| `/edit_rank` | `роль`, `new_required_xp` | Изменяет порог XP для существующего звания. |
| `/remove_rank` | `роль` | Удаляет звание из БД и **физически снимает роль со всех участников**. |
| `/sync_ranks` | — | Принудительно пересчитывает и синхронизирует звания у **всего состава клана**. Используется после массовых изменений в системе ачивок или при ручной правке ролей в Discord. |
| `/check_profile` | `member` | Просматривает XP-профиль и текущее звание конкретного бойца. Перед выводом принудительно пересчитывает звание. |

#### Управление активностью

| Команда | Параметры | Описание |
|---|---|---|
| `/link_user` | `member`, `steam_id`, `squad_nickname` | Привязывает Discord-аккаунт бойца к Steam ID. |
| `/edit_link` | `member`, `new_steam_id`, `new_squad_nickname` | Изменяет существующую привязку. |
| `/unlink_user` | `member` | Удаляет привязку пользователя из базы. |
| `/check_user` | `member`, `[period]` | Детальная статистика активности конкретного бойца: Бой / Сидинг / Тренировки / топ серверов за выбранный период. |
| `/export_stats` | — | Генерирует HTML-отчёт со статистикой **всего состава** (Бой / Сидинг / Тренировки / Всего / все серверы по убыванию). Отправляется как скачиваемый файл. |

### Команды Клана

*Доступны всем участникам с ролью `CLAN_ROLE_ID`.*

#### XP и звания

| Команда | Параметры | Описание |
|---|---|---|
| `/profile` | — | Карточка профиля: текущее звание, накопленный опыт и визуальный прогресс-бар до следующего ранга. |
| `/ranks_list` | — | Таблица всех воинских званий и необходимого XP для каждого из них, отсортированная по возрастанию порога. |
| `/achievements_list` | `[category]` | Список всех достижений с XP и описанием, сгруппированный по категориям. Если передана `category` — показывает только её. Поддерживает автодополнение по названиям существующих категорий. |

#### Статистика активности

*Все команды поддерживают опциональный параметр `[period]`: 1 день, 3 дня, 7 дней, 14 дней, 1 месяц, 6 месяцев, 1 год. По умолчанию — за всё время.*

| Команда | Параметры | Описание |
|---|---|---|
| `/my_stats` | `[period]` | Личная статистика: Бой / Сидинг / Тренировки / любимые серверы за выбранный период. |
| `/top_players` | `[period]` | ТОП-10 самых активных бойцов клана по суммарному времени в игре. |
| `/top_servers` | `[period]` | ТОП-10 серверов, на которых чаще всего играет состав (учитываются все три категории). |
| `/clan_stats` | `[period]` | Глобальная статистика клана: суммарное время в боях, на сидинге и тренировках по всем активным бойцам, а также количество активных игроков за период. |
