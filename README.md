# Discord XP & Ranks Bot

Этот бот автоматизирует систему опыта и званий. Опыт высчитывается «на лету» на основе ролей-ачивок, которые есть у пользователя. При достижении определенного количества опыта бот автоматически выдает новое звание и забирает старое.

## Стек технологий
- **Python 3.8+**
- **discord.py** (фреймворк для Discord API)
- **MongoDB** + **Motor** (асинхронная база данных)

---

## 1. Настройка в Discord Developer Portal

Перед запуском бота его необходимо правильно настроить на [портале разработчиков](https://discord.com/developers/applications).

1. Создайте новое приложение.
2. Во вкладке **Bot**:
   - Отключите ползунок `Requires OAuth2 Code Grant`.
   - В разделе **Privileged Gateway Intents** ОБЯЗАТЕЛЬНО включите ползунок `Server Members Intent` (без него бот не сможет отслеживать изменения ролей у пользователей).
3. Во вкладке **OAuth2 -> URL Generator**:
   - В блоке **Scopes** выберите: `bot` и `applications.commands`.
   - В блоке **Bot Permissions** выберите: `Administrator` (или вручную выдайте `Manage Roles`, `Send Messages`, `Embed Links`, `Use Slash Commands`).
4. Скопируйте сгенерированную ссылку в самом низу и добавьте бота на свой сервер.

> **КРИТИЧЕСКИ ВАЖНО:** После добавления бота на сервер зайдите в Настройки Сервера -> Роли. Переместите роль вашего бота **В САМЫЙ ВЕРХ СПИСКА**. Если роль бота будет ниже ролей званий, бот не сможет их выдавать (Discord выдаст ошибку `Forbidden`).

## 2. Установка и запуск

1. Убедитесь, что у вас установлен Python и локально запущена MongoDB (стандартный порт `27017`).
2. Откройте терминал в папке проекта и установите зависимости:
   ```bash
   pip install discord.py motor python-dotenv
   ```
3. Создайте файл `.env` в корневой папке проекта и добавьте туда ваши данные:
   ```env
   DISCORD_TOKEN=ВАШ_ТОКЕН_БОТА_С_ПОРТАЛА
   MONGO_URI=mongodb://localhost:27017/
   ```
4. Запустите бота:
   ```bash
   python main.py
   ```
   *Если всё сделано верно, в консоли появится сообщение: `✅ Бот запущен! Подключение к MongoDB установлено.`*

## 3. Как это работает (Архитектура)

Опыт **не хранится** в базе данных как отдельная цифра. База данных используется только для хранения "Настроек сервера" (сколько опыта дает какая роль). 
Когда модератор выдает или забирает роль-ачивку у пользователя, бот:
1. Замечает изменение ролей.
2. Суммирует ценность всех ачивок пользователя.
3. Находит подходящее звание в БД.
4. Синхронизирует роли: жестко забирает все неактуальные звания и выдает нужное.

> **Логирование:** Все действия бота (выдача ролей, ошибки БД, добавление ачивок) автоматически записываются в файл `bot.log` в папке проекта.

## 4. Список команд

Все команды реализованы через современные Slash-команды (`/`).

### Команды Администратора
Эти команды видят и могут использовать только пользователи с правами Администратора на сервере. Ответы бота на эти команды скрыты от обычных пользователей.

* `/add_achievement [роль] [xp] [описание]`
  Регистрирует Discord-роль в базе данных как "Ачивку". Устанавливает, сколько опыта она дает при получении.
* `/remove_achievement [роль]`
  Удаляет роль из списка ачивок в базе данных.
* `/add_rank [роль] [required_xp]`
  Регистрирует Discord-роль как "Звание". Устанавливает порог опыта, необходимый для автоматического получения этой роли.
* `/remove_rank [роль]`
  Удаляет роль из списка званий.

### Команды Пользователей
* `/profile`
  Выводит красивую карточку профиля (Embed). Показывает текущее звание пользователя, количество накопленного опыта и визуальный прогресс-бар до следующего звания.
