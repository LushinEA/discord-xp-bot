# Squad Clan Discord Bot

Многофункциональный Discord-бот для управления кланом в игре **Squad**. 
Бот состоит из двух независимых систем:
1. **Система опыта и званий (XP & Ranks)** — автоматизирует выдачу званий на основе ролей-ачивок.
2. **Трекер активности (Activity Tracker)** — отслеживает, сколько времени участники клана проводят в игре, автоматически разделяя время на "Бой" (Battle) и "Сидинг" (Seeding). Ведет детальный учет по серверам, составляет топы игроков и поддерживает **фильтрацию статистики по периодам** (день, неделя, месяц, год).

---

## Стек технологий
- **Python 3.9+**
- **discord.py** (фреймворк для Discord API)
- **MongoDB** + **Motor** / **Pymongo** (асинхронная и синхронная работа с БД)

## 1. Настройка в Discord Developer Portal

Перед запуском бота его необходимо правильно настроить на [портале разработчиков](https://discord.com/developers/applications).

1. Создайте новое приложение.
2. Во вкладке **Bot**:
   - Отключите ползунок `Requires OAuth2 Code Grant`.
   - В разделе **Privileged Gateway Intents** ОБЯЗАТЕЛЬНО включите два ползунка:
     - `Server Members Intent` (нужен для отслеживания изменения ролей-ачивок).
     - `Presence Intent` (нужен для отслеживания статуса "Играет в Squad").
3. Во вкладке **OAuth2 -> URL Generator**:
   - В блоке **Scopes** выберите: `bot` и `applications.commands`.
   - В блоке **Bot Permissions** выберите: `Administrator` (или вручную выдайте `Manage Roles`, `Send Messages`, `Embed Links`, `Use Slash Commands`).
4. Скопируйте сгенерированную ссылку в самом низу и добавьте бота на свой сервер.

> **КРИТИЧЕСКИ ВАЖНО:** После добавления бота на сервер зайдите в Настройки Сервера -> Роли. Переместите роль вашего бота **В САМЫЙ ВЕРХ СПИСКА**. Если роль бота будет ниже ролей званий, бот не сможет их выдавать (Discord выдаст ошибку `Forbidden`).

## 2. Установка и запуск

1. Убедитесь, что у вас установлен Python и локально запущена MongoDB (стандартный порт `27017`).
2. Откройте терминал в папке проекта и установите зависимости:
   ```bash
   pip install discord.py motor pymongo python-dotenv
   ```
3. Создайте файл `.env` в корневой папке проекта и добавьте туда ваши настройки:
   ```env
   DISCORD_TOKEN=ВАШ_ТОКЕН_БОТА
   MONGO_URI=mongodb://localhost:27017/
   STEAM_API_KEY=ВАШ_КЛЮЧ_STEAM_API

   # ID роли клана (бот будет отслеживать активность только пользователей с этой ролью)
   CLAN_ROLE_ID=111111111111111111

   # ID ролей администраторов бота (указывать через запятую без пробелов)
   ADMIN_ROLE_IDS=222222222222222222,333333333333333333
   ```
4. Запустите бота:
   ```bash
   python main.py
   ```

## 3. Массовый импорт состава (import_users.py)

Чтобы не привязывать каждого бойца вручную, предусмотрен скрипт для заливки таблицы из Excel/Google Sheets.

1. Откройте файл `users_data.txt`.
2. Вставьте туда данные бойцов. Формат: 4 колонки, разделенные `Tab` (стандарт при копировании из таблиц):
   `[Ник в Squad]   [Steam ID]   [Discord Ник]   [Discord ID]`
3. Запустите скрипт:
   ```bash
   python import_users.py
   ```
Скрипт свяжет Discord ID с игровым ником и Steam ID. База готова к трекингу!

## 4. Как это работает (Архитектура)

### Система опыта и званий (XP System)
Опыт не хранится в виде цифры, а считается "на лету" из ценности выданных пользователю ролей-ачивок.
1. Бот реагирует на выдачу/снятие роли модератором.
2. Проверяет наличие `CLAN_ROLE_ID` у пользователя.
3. Встроена **перекрёстная защита**: роль не может быть одновременно ачивкой и званием.
4. При удалении ачивки из БД бот **автоматически снимает её со всех участников** и пересчитывает их звания по цепочке.

### Трекер активности (Squad Tracker)
Трекер работает по принципу *Stateless 1-minute loop* с использованием *BulkWrite* и системы **Daily Buckets** (Ежедневных корзин). В цикл встроены асинхронные паузы для защиты от зависаний (ошибка 10062) при большом онлайне.
1. Каждую минуту бот проверяет Rich Presence тех пользователей, у которых есть `CLAN_ROLE_ID` и которые занесены в БД.
2. Разделяет время: меньше 50 человек на сервере — **"Сидинг"**, больше — **"Бой"**.
3. Сохраняет данные в MongoDB двумя независимыми потоками:
   - В глобальную коллекцию (для статистики за всё время).
   - В ежедневную коллекцию (с ключом `Steam ID + Дата`).
   Это гарантирует 100% сохранность статистики при сбоях и позволяет боту мгновенно генерировать топы за любой исторический период с помощью *MongoDB Aggregation Pipelines*.

## 5. Список команд (Slash Commands)

### Команды Администратора
*Доступны только пользователям с ролями, указанными в `ADMIN_ROLE_IDS` в файле `.env`.*

**Управление XP:**
* `/add_achievement [роль] [xp] [описание]` — Регистрирует роль как "Ачивку" с ценностью опыта.
* `/edit_achievement [роль] [new_xp] [new_description]` — Изменяет параметры существующей ачивки.
* `/remove_achievement [роль]` — Удаляет роль из базы **и физически снимает её со всех участников**.
* `/add_rank [роль] [required_xp]` — Регистрирует роль как "Звание" с порогом выдачи.
* `/edit_rank [роль] [new_required_xp]` — Изменяет порог опыта для существующего звания.
* `/remove_rank [роль]` — Удаляет звание из базы **и физически снимает его со всех участников**.

**Управление активностью:**
* `/link_user [user] [steam_id] [squad_nick]` — Вручную привязывает Discord-аккаунт бойца к его Steam ID.
* `/edit_link [user] [new_steam_id] [new_squad_nick]` — Изменяет существующую привязку бойца.
* `/unlink_user [user]` — Удаляет привязку пользователя из базы.
* `/check_user [user] [period]` — Выводит подробную статистику онлайна конкретного бойца за выбранный период.
* `/export_stats` — Генерирует и отправляет скачиваемый HTML-отчет со статистикой активности всего состава клана.

### Команды Клана
*Доступны всем участникам с ролью `CLAN_ROLE_ID`. Большинство команд поддерживают опциональный параметр `[period]` (1 день, 7 дней, месяц и т.д.). По умолчанию выводится статистика за всё время.*

* `/profile` — Карточка профиля: текущее звание, опыт и визуальный прогресс до следующего ранга.
* `/my_stats [period]` — Личная статистика в Squad за выбранный период: время в боях, сидинге и любимые серверы.
* `/top_players [period]` — Выводит ТОП-10 самых активных бойцов клана за указанное время.
* `/top_servers [period]` — Выводит ТОП-10 серверов, на которых чаще всего играет состав клана.
* `/clan_stats [period]` — Глобальная статистика клана: сумма всех часов в бою и на сидинге по всем активным игрокам за период.
